# ONNX MNIST 手書き数字分類

ONNX Runtimeを使用して、事前学習済みのMNISTモデルで画像から手書き数字（0-9）を分類するC#コンソールアプリケーションです。

## 機能

- 手書き数字が含まれる画像（JPG/PNG）の読み込みと処理
- MNIST互換のために画像を28x28ピクセルにリサイズし、色を反転
- ONNX Runtimeを使用した効率的な推論
- ソフトマックス関数を適用してロジットを確率に変換
- 信頼度スコア付きで分類結果を表示

## 要件

- .NET 10.0以降
- ONNXモデルファイル: `mnist-8.onnx`
- テスト画像: `0.jpg` から `9.jpg`（またはカスタム画像）

## 依存パッケージのインストール

ビルド前に以下のコマンドを実行して必要なパッケージをインストールしてください：

```bash
dotnet add package Microsoft.ML.OnnxRuntime --version 1.26.0
dotnet add package SixLabors.ImageSharp --version 2.1.6
```

## ビルド

```bash
dotnet build
```

## 実行

```bash
dotnet run
```

アプリケーションは `Program.cs` の `imagePaths` リストで指定されたすべての画像を処理し、分類結果を表示します。

## 仕組み

1. **画像読み込み**: ImageSharpを使用してグレースケール（L8形式）で画像を読み込み
2. **前処理**:
   - 画像を28x28ピクセルにリサイズ
   - 色を反転（背景が黒、文字が白になるように）
   - ピクセル値を0.0-1.0の範囲に正規化
3. **推論**:
   - ONNX入力用の4次元テンソル [1, 1, 28, 28] を作成
   - MNIST-8 ONNXモデルを使用して推論を実行
4. **後処理**:
   - ソフトマックス関数を適用して生のロジットを確率に変換
   - 最も確率の高い数字を特定
5. **出力**: 予測された数字と信頼度をパーセンテージで表示

## サンプル出力

```
【AI判定結果】 画像に書かれた数字は 「 5 」 です！ (確信度: 99.8% image: 5.jpg)
```

## カスタマイズ

独自の画像を使用する場合：
1. プロジェクトディレクトリに画像ファイルを追加
2. `Program.cs` の `imagePaths` リストをファイル名で更新
3. 画像が明るい背景に明確な手書き数字であることを確認

## モデル

このアプリケーションは `mnist-8.onnx` モデルを使用しています。これはMNIST数字分類用の事前学習済みニューラルネットワークです。

ONNXについての詳細はこちら: [https://onnx.ai/](https://onnx.ai/)
